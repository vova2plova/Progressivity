# Чеклист завершения задачи

После выполнения любой задачи (особенно редактирования кода) необходимо выполнить следующие проверки:

## 1. Backend проверки
```bash
# Запустить линтеры и проверки кода
make check
# Эта команда выполняет:
# - go vet ./... (проверка корректности кода)
# - golangci-lint run ./... (расширенный линтинг)
# - go test ./... -v -count=1 (запуск всех тестов)

# Если тесты падают, исправить их перед продолжением
```

## 2. Frontend проверки
```bash
# Перейти в директорию фронтенда
cd web

# Запустить линтер
npm run lint

# Если есть ошибки линтера, исправить их
npm run lint:fix

# Форматирование кода
npm run format

# Запустить TypeScript компиляцию (проверка типов)
npx tsc --noEmit
```

## 3. Интеграционные проверки
```bash
# Вернуться в корень проекта
cd ..

# Проверить, что миграции работают
make migrate-down 1 && make migrate-up

# Запустить приложение и проверить базовый функционал
make run
# В другом терминале: cd web && npm run dev
```

## 4. Перед коммитом
- [ ] Все тесты проходят (`make test`)
- [ ] Линтер не показывает ошибок (`make lint`, `npm run lint`)
- [ ] Код отформатирован (`npm run format`, golangci-lint включает gofmt)
- [ ] TypeScript компилируется без ошибок
- [ ] Миграции применены и работают
- [ ] Не добавлены секреты в коммит (.env, credentials и т.д.)

## 5. Дополнительные проверки для определённых изменений

### При изменении бэкенда:
- [ ] Написаны unit-тесты для новой бизнес-логики
- [ ] Обновлены существующие тесты при изменении интерфейсов
- [ ] Проверена работа API через curl или Postman

### При изменении фронтенда:
- [ ] Компоненты работают в изоляции (Storybook не используется, но можно проверить вручную)
- [ ] Нет предупреждений React (ключи списков, хуки)
- [ ] Адаптивный дизайн (проверить на разных размерах экрана)

### При изменении схемы БД:
- [ ] Создана миграция (up и down)
- [ ] Миграция протестирована (up → down → up)
- [ ] Не потеряны данные (проверить down-миграцию)

## 6. Запуск полного пайплайна
```bash
# Полный цикл проверки (рекомендуется перед пул-реквестом)
make check && cd web && npm run lint && npx tsc --noEmit && cd .. && make migrate-down 1 && make migrate-up
```

## 7. Особенности Windows окружения
- Использовать Git Bash или WSL для выполнения команд Makefile
- Docker должен быть запущен для работы с БД
- Убедиться, что переменные окружения установлены (.env файл)
